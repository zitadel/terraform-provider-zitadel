# zitadel_organization_domain (Resource)

Resource representing a domain of an organization in ZITADEL.

## Example Usage

```terraform
resource "zitadel_organization_domain" "default" {
  organization_id = zitadel_organization.default.id
  domain          = "example.com"
  validation_type = "DOMAIN_VALIDATION_TYPE_DNS"
}

output "dns_validation_token" {
  value     = zitadel_organization_domain.default.validation_token
  sensitive = true
}

resource "zitadel_organization_domain" "verified" {
  organization_id = zitadel_organization.default.id
  domain          = "verified.example.com"
  validation_type = "DOMAIN_VALIDATION_TYPE_HTTP"
  verify          = true
}
```

## Schema

### Required

- `domain` (String) Domain name to be added to the organization
- `organization_id` (String) ID of the organization
- `validation_type` (String) Type of domain validation, supported values: DOMAIN_VALIDATION_TYPE_UNSPECIFIED, DOMAIN_VALIDATION_TYPE_HTTP, DOMAIN_VALIDATION_TYPE_DNS

### Optional

- `verify` (Boolean) Trigger domain verification. Set to true after adding DNS/HTTP validation.

### Read-Only

- `id` (String) The ID of this resource.
- `is_primary` (Boolean) Whether this is the primary domain of the organization
- `is_verified` (Boolean) Whether the domain has been verified
- `validation_token` (String, Sensitive) Validation token for domain verification
- `validation_url` (String) URL where validation file should be hosted for HTTP verification
